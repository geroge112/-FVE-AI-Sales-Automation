Project Objective

Automate standard FVE sales without losing operational control.

The system starts when a lead is imported into Odoo. From that moment, AI handles qualification, customer communication and proposal preparation, while deterministic services handle pricing, validation and workflow updates.

Key Principle
AI communicates, system services calculate, validation rules control risk and Odoo remains the system of record.

AI-led Communication

Initial contact, qualification, FAQ handling, objection handling, follow-up and customer confirmation.

Deterministic Calculation

FVE sizing, product variants, pricing, margin limits and discount rules are calculated by controlled services.

Human Fallback

Low confidence, non-standard roofs, pricing exceptions, duplicate leads and GDPR-sensitive cases are escalated.

Architecture

Odoo-first architecture with a dedicated workflow and decision layer.

The AI Orchestrator coordinates the bot, knowledge retrieval, pricing, FVE configuration, document generation and Odoo updates.

Architecture Principles

Design principles keep the automation useful, controlled and maintainable.

These rules define where AI is allowed to help, where deterministic systems remain responsible and how the workflow stays auditable.

AI is the communication layer, not the source of truth.The AI Sales Bot explains, asks and summarizes. It does not own price lists, technical limits or contractual rules.
Critical logic remains deterministic.Pricing, discounts, margins, package selection and approval flags are handled by controlled services.
Human escalation is part of the design.Low-confidence, non-standard, high-value or compliance-sensitive cases are routed to a human owner with context.
Odoo remains the operational core.Leads, statuses, offers, orders, tasks and audit records are stored in Odoo as the primary system of record.
Integrations are event-driven.Lead creation, customer replies, uploads and confirmations trigger workflow events instead of manual follow-up.
Auditability by default.Important decisions store input data, confidence score, service output, selected action and escalation reason.
System Modules

Each module has a clear responsibility, inputs, outputs and delivery scope.

The implementation is intentionally modular so that MVP can be launched safely and expanded later.

Odoo CRM

System of record for leads, pipeline, offers, orders and audit history.

Inputs

  • Imported lead data
  • AI qualification updates
  • Offer and order data

Tasks

Configure CRM fields, pipeline stages, webhook triggers, API service user, escalation views and AI activity log.

Deliverables

  • AI-enabled CRM pipeline
  • Escalation queue
  • Audit visibility

AI Sales Bot

Customer-facing layer for qualification, explanation and follow-up.

Conversation Scope

  • Initial contact
  • Qualification questions
  • Offer explanation
  • Follow-up and handoff

Implementation Focus

Build structured extraction, intent detection, missing-data handling, safe RAG answering and channel adapters.

Outputs

  • Qualified customer profile
  • Conversation summary
  • Next-best workflow action

AI Orchestrator

Coordinates workflow state, service calls, validation and Odoo updates.

Core Functions

  • State machine
  • Tool routing
  • Confidence scoring
  • Escalation decisions

Resilience

Implement resilient orchestration mechanisms including retries, idempotency, timeouts, structured logging and monitoring.

Deliverables

  • Webhook listener
  • Workflow engine
  • Audit event stream

RAG Knowledge Base

Approved knowledge layer for grounded AI responses.

Sources

  • Product sheets
  • FAQ and sales scripts
  • Warranty and legal text
  • Installation process docs

Tasks

Normalize documents, chunk content, add metadata, build vector index, version approved content and define forbidden-topic escalation.

Rules

  • Use approved sources
  • Track content version
  • Escalate unsupported topics

OCR & Multimodal AI

Extracts structured data from invoices and customer-provided documents.

Extracted Fields

  • Annual consumption
  • Tariff and billing period
  • Address and customer name
  • Monthly or yearly cost

Validation

Validate numeric consumption, address consistency, billing period and OCR confidence before using extracted data for pricing.

Fallback

Ask the customer to confirm uncertain values or escalate the invoice to manual review when extraction confidence is low.

FVE Configurator & Pricing

Produces technical-commercial proposal and validated price.

Configurator Outputs

  • Recommended kWp
  • Panel and inverter variant
  • Battery and wallbox option
  • Technical risk flags

Pricing Rules

Apply price lists, package rules, margin thresholds, discount limits, offer validity and approval-required flags.

Deliverables

  • Structured proposal
  • Price breakdown
  • Approval flag if needed

Operational Risk

Production automation expects incomplete data, broken APIs and uncertain AI outputs.

The system is designed to continue safely, ask for clarification or escalate instead of silently producing incorrect proposals.

Operational Risks

API failures Incorrect OCR Low AI confidence Duplicate leads GDPR compliance
API fail → Retry queue → Alert → Escalation

Mitigation Controls

Retry queue Human escalation Audit logs Validation rules Duplicate matching
>85% auto / 60-85% clarify / <60% handoff
Escalation Trigger
Confidence score above 85% continues automatically, 60-85% asks a clarification question, and below 60% creates a human escalation task in Odoo.
